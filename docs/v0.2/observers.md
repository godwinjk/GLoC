# Observers — v0.2

Every reactor generated by `#[reactor]` includes an `on_change()` method that
lets you register callbacks to be notified on every real state transition.

---

## What is an observer?

An observer is a closure that fires **synchronously** inside `emit()`,
immediately after the state has been updated, but only when the state
actually changed (change-detection still applies).

```
emit(next)
    │
    ├─ next == current? → return (no-op)
    │
    ├─ update __gloc_state = next
    │
    └─ call each registered observer with &new_state
```

---

## Basic usage

```rust
use gloc::{reactor, Reactor};

#[derive(Clone, PartialEq, Debug)]
pub struct ScoreState { pub value: u32 }

#[reactor(state = ScoreState)]
pub struct ScoreReactor {}

impl ScoreReactor {
    pub fn add(&mut self, points: u32) {
        let next = self.state().value + points;
        self.emit(ScoreState { value: next });
    }
}

let mut reactor = ScoreReactor::new(ScoreState { value: 0 });

// Register an observer
reactor.on_change(|state| {
    println!("Score changed → {}", state.value);
});

reactor.add(10);  // prints: Score changed → 10
reactor.add(5);   // prints: Score changed → 15
```

## Multiple observers

You can register as many observers as you need.
They fire in registration order:

```rust
reactor.on_change(|state| println!("[Logger] score = {}", state.value));
reactor.on_change(|state| {
    if state.value >= 100 {
        println!("[Achievement] score milestone reached!");
    }
});
reactor.on_change(|state| analytics::track("score_change", state.value));

reactor.add(50);
// [Logger] score = 50
reactor.add(60);
// [Logger] score = 110
// [Achievement] score milestone reached!
```

## Observers and change detection

Observers are **only called when state actually changes**.
Emitting the same value is still a no-op — no observers fire:

```rust
let mut reactor = ScoreReactor::new(ScoreState { value: 5 });
let mut call_count = 0;

reactor.on_change(|_| call_count += 1);

// Emit the same value — no observer fires
reactor.emit(ScoreState { value: 5 });
reactor.emit(ScoreState { value: 5 });

assert_eq!(call_count, 0);

// Emit a different value — observer fires once
reactor.emit(ScoreState { value: 10 });
assert_eq!(call_count, 1);
```

## Capturing state in observers

Observers receive a shared reference `&StateType` to the new state.
To capture values for later use, clone inside the closure:

```rust
use std::sync::{Arc, Mutex};

let log: Arc<Mutex<Vec<u32>>> = Arc::new(Mutex::new(vec![]));
let log_clone = log.clone();

reactor.on_change(move |state| {
    log_clone.lock().unwrap().push(state.value);
});

reactor.add(10);
reactor.add(20);

assert_eq!(*log.lock().unwrap(), vec![10, 30]);
```

## Common use cases

### UI re-rendering (Dioxus)

```rust
// In a Dioxus app, the signal write triggers re-render automatically.
// on_change is used when you need a side effect on transition:
reactor.on_change(|state| {
    if state.is_authenticated {
        navigate_to("/dashboard");
    }
});
```

### Analytics

```rust
reactor.on_change(|state| {
    analytics::track("state_transition", json!({ "count": state.count }));
});
```

### Logging

```rust
reactor.on_change(|state| {
    tracing::info!(count = state.count, "CounterReactor transitioned");
});
```

### Persistence

```rust
reactor.on_change(|state| {
    if let Err(e) = save_to_disk(state) {
        eprintln!("Failed to persist state: {e}");
    }
});
```

## Suppressing observers

If you don't need `on_change()` — for example in a backend service or CLI
where there is no UI to notify — suppress it with `no_observers`:

```rust
#[reactor(state = JobState, no_observers)]
pub struct JobCubit {}
```

This also removes the `__gloc_listeners` field from the struct, making
the generated reactor slightly smaller.
